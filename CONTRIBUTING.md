# 参与贡献

## 标准流程

1. 在 GitHub Issue 中认领任务并明确验收标准。
2. 从最新 `main` 创建分支，例如 `feat/task-state-machine`。
3. 完成功能、文档和必要测试。
4. 提交 Pull Request，关联 Issue，并邀请至少一名成员审阅。
5. 通过测试和审阅后合并，删除功能分支。

## 完成定义

- 功能符合 Issue 的验收标准
- 输入输出遵循已发布的数据格式版本
- 新逻辑包含测试或可复现的人工测试步骤
- 不提交密钥、串口日志中的隐私数据、构建产物或编辑器缓存
- 相关设计和使用说明已同步更新

## 冲突处理

接口或协议变更必须先开 Issue 讨论，由项目负责人确认版本号和迁移方式。影响其他模块时，不应只在代码中静默修改。
