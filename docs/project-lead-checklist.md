# 项目负责人检查清单

## 每周

- 整理并分配 GitHub Issues
- 检查 Pull Requests、接口变更和模块阻塞
- 组织一次集成测试，记录环境、版本、步骤、结果和责任人
- 更新里程碑风险与下周目标

## 每次联调前

- 冻结本次使用的数据格式和通信协议版本
- 确认各模块分支已合并且自动化测试通过
- 准备测试数据、设备、线缆和回退版本

## 每次联调后

- 将缺陷拆为 Issues 并设置负责人和截止时间
- 保存必要日志与测试结论，不提交敏感信息
- 更新接口文档和状态机说明

## 结题材料

- 需求与总体架构
- 关键模块和接口说明
- 测试方案、数据与结果
- 项目过程、版本发布与成员贡献
- 演示视频、使用手册和已知限制
