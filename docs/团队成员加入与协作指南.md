# 手写机器人二层开发：团队成员加入与协作指南

## 一、项目地址

GitHub 仓库：

<https://github.com/123telamasipu/handwriting-robot-layer2>

项目目前为公开仓库，任何人可以查看和 Fork；代码合并仍由项目负责人审核。

## 二、第一次加入项目

1. 注册并登录 GitHub。
2. 打开项目地址，点击右上角 **Fork**，将仓库复制到自己的 GitHub 账号。
3. 打开自己的 Fork 仓库，点击 **Code → HTTPS**，复制仓库地址。
4. 在电脑终端执行：

   ```bash
   git clone 你的Fork仓库地址
   cd handwriting-robot-layer2
   ```

5. 确认自己负责的模块：

   - `ljy`：`src/handwriting_robot_layer2_ljy_integration/`
   - `wl`：`src/handwriting_robot_layer2_wl_handwriting/`
   - `nrx`：`src/handwriting_robot_layer2_nrx_vectorization/`
   - `cx`：`src/handwriting_robot_layer2_cx_ui/`
   - `bxy`：`src/handwriting_robot_layer2_bxy_device/`

## 三、开始一次开发

先同步主分支，再创建自己的功能分支：

```bash
git switch main
git pull origin main
git switch -c feat/成员缩写-功能名称
```

例如：

```bash
git switch -c feat/wl-handwriting-scan
```

只在自己的模块和相关测试中修改代码，不要直接修改其他成员模块，也不要直接提交 `main`。

## 四、安全提交代码

提交前检查：

- 没有密码、Token、账号、隐私照片或未经脱敏的实验数据；
- 没有大型视频、原始扫描件或构建缓存；
- 输入输出符合 `docs/stroke-interface.md`；
- 已运行相关测试；
- README 或接口说明已同步更新。

提交并推送：

```bash
git status
git add 需要提交的文件
git commit -m "feat: 简短描述本次改动"
git push -u origin feat/成员缩写-功能名称
```

不要使用 `git add .` 提交自己没有检查过的文件。

## 五、提交 Pull Request

打开自己的 Fork 仓库，点击 **Compare & pull request**，写清改动内容、测试输入、测试结果、对其他模块的影响和剩余问题，然后发给成员1审核。不要自行合并到 `main`。

## 六、每周进度汇报格式

```text
成员：
本周完成：
测试结果：
遇到的问题：
需要谁协助：
下周计划：
```

## 七、最重要的规则

- 所有模块通过统一 `StrokeDocument` / `strokes` 接口协作；
- 成员2不直接控制设备，成员3不单独制作界面；
- `main` 分支保持可运行，所有改动通过 Pull Request 合并；
- 不确定是否能改动其他模块时，先开 Issue 讨论。
