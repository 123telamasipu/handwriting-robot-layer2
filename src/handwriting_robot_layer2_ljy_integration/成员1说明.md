# 成员1（ljy）：项目负责人兼系统集成

## 主要职责

### （1）项目总体管理
- 制定项目整体开发计划和阶段性目标；
- 协调各成员工作，合理分配开发任务；
- 组织项目例会，跟踪项目进度，及时解决开发过程中出现的问题；
- 负责项目风险控制和整体推进。

### （2）系统总体设计与集成
- 负责写字机二次开发的软件总体架构设计；
- 制定各功能模块之间的接口规范和数据交互格式；
- 完成各子模块的集成、联调及系统整体维护；
- 保证各模块稳定协同运行，实现完整的软件工作流程。

### （3）设备选型与二次开发平台搭建
- 调研并采购满足项目需求的写字机设备；
- 制定设备采购技术指标及测试方案；
- 验证设备的开放性、通信方式及二次开发能力；
- 搭建整体开发与测试环境，为后续软件开发提供平台支持。

### （4）GitHub仓库与版本管理
- 建立并维护项目 GitHub 仓库；
- 制定代码管理规范，包括分支管理、提交规范和版本控制；
- 完成代码合并、版本发布及开发文档维护，保证多人协作开发的规范性。

### （5）设备通信与控制程序开发
- 负责设备通信模块开发，完成上位机与写字机之间的数据交互；
- 设计并实现统一的数据接口及控制协议；
- 实现设备连接、任务下发、状态反馈等核心控制功能。

### （6）任务调度与状态机设计
- 设计系统整体运行流程；
- 实现任务状态机，对设备运行状态进行统一管理；
- 实现任务启动、暂停、继续、结束及异常处理等功能，保证系统运行稳定可靠。

### （7）系统测试与集成验证
- 制定系统测试计划；
- 组织各模块联调和阶段性测试；
- 分析测试结果，定位并协调解决系统问题；
- 验证系统功能、稳定性及整体性能。

### （8）项目文档与成果整理
- 汇总项目开发过程中的技术资料；
- 编写项目设计文档、开发说明及测试报告；
- 负责中期检查、结题验收及成果展示材料的整理与汇报。

## 工作边界

成员1负责总体架构、模块接口和系统集成，不替代其他成员实现笔迹算法、矢量化算法或界面功能。接口变更必须先在 Issue 或 Pull Request 中说明。

## 安全提交与进度汇报

1. 从最新 `main` 创建 `feat/ljy-integration-日期-主题` 分支。
2. 只修改 `src/handwriting_robot_layer2_ljy_integration/` 和相关文档、测试。
3. 提交前运行测试，提交信息使用 `feat:`、`fix:` 或 `docs:` 开头。
4. 推送分支后创建 Pull Request，写清完成内容、测试结果和待办事项。
5. 每周在 Issue 中汇报：已完成、测试结果、风险、下周计划。

禁止提交密码、Token、个人隐私、完整设备日志和无关生成文件。
