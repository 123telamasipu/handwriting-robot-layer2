# 手机笔迹采集功能说明

## 用途

手机笔迹采集功能用于在没有数位板时，通过手机浏览器录入单字轨迹。笔记本运行局域网服务，手机只负责触控输入；样本最终仍由笔记本上的 `SampleStore` 保存，因此和桌面鼠标、数位板采集共用同一份个人字符库。

该服务不连接互联网，不控制写字机，也不负责页面排版。

## 启动条件

- 手机和笔记本连接同一个 Wi-Fi 或手机热点。
- 笔记本安装 Python 3.9 或更高版本。
- Windows 防火墙允许 Python 在“专用网络”中通信。
- 端口 `8765` 未被其他程序占用；也可以通过 `--port` 修改。

手机采集服务只使用 Python 标准库，不依赖 PySide6，也不需要安装 Node.js。

## 启动方法

在仓库根目录双击或运行：

```powershell
src\handwriting_robot_layer2_wl_handwriting\tools\run_mobile_collector.cmd `
  --writer-id user_01 `
  --writer-name "测试用户"
```

也可以直接运行 Python 模块：

```powershell
py -3 -m src.handwriting_robot_layer2_wl_handwriting.mobile_server `
  --writer-id user_01
```

启动后终端会显示类似地址：

```text
笔记本访问: http://127.0.0.1:8765/?token=123456
手机访问:   http://192.168.0.104:8765/?token=123456
```

在手机浏览器中输入“手机访问”地址。必须保留 `?token=...` 部分，否则服务会拒绝访问。终端需要保持打开，按 `Ctrl+C` 停止服务。

## 已完成功能

- 书写者选择与本机浏览器记忆。
- 1140 个目标字符的顺序浏览、序号跳转和字符查找。
- 同一字符 1～5 个变体切换。
- 手指、手机触控笔和鼠标 Pointer Events 采集。
- 浏览器合并轨迹点采集，保留压力、倾角、旋转和时间字段。
- 每个笔画结束后自动保存草稿。
- 撤销上一笔、清空当前画布、正式保存和保存后继续。
- 采集进度、笔画数、轨迹点数和输入来源显示。
- 断点续录：重新打开页面后恢复草稿或正式样本。

## 手机录入流程

1. 在笔记本启动服务并保持终端打开。
2. 用手机打开终端显示的局域网地址。
3. 确认书写者编号和备注，进入采集页面。
4. 在米字格中书写当前字符；每次落笔到抬笔保存为一个笔画。
5. 使用撤销或清空修正错误。
6. 点击“保存并继续”，进入下一未完成字符。

## 数据位置与隐私

默认数据目录为：

```text
runtime_data/handwriting/writers/<writer_id>/
```

该目录已被 Git 忽略。手机页面会在浏览器 `localStorage` 中保存书写者编号和备注，方便下次继续；共用手机时应清理该网站的浏览器数据。

服务默认生成六位访问令牌，用于阻止同一局域网中的随意访问。HTTP 传输没有加密，因此只能在可信的家庭、实验室专用 Wi-Fi 或个人热点中使用，不应开放到公网。除非是隔离的测试环境，不要使用 `--no-auth`。

## 常见问题

### 手机无法打开地址

1. 确认手机和笔记本位于同一网络。
2. 使用 WLAN 对应的 `192.168.*.*` 地址，不要使用 `127.0.0.1`、VMware 或其他虚拟网卡地址。
3. 在 Windows 防火墙提示中只允许“专用网络”。
4. 执行 `ipconfig` 确认笔记本 WLAN IPv4 地址。
5. 尝试关闭服务后改用其他端口：`--port 8877`。

### 轨迹出现断笔

关闭浏览器页面缩放、系统手势或悬浮窗，再重新录入。触控笔应直接落在画布内部。不同手机对压力和倾角的支持不同，不支持的字段会使用中性默认值。

### 正式样本没有被清空

“清空”只清除当前画布和草稿，不删除已经正式保存的样本。清空后重新书写并点击保存，会覆盖同一字符和变体的正式样本。
