# 用户笔迹样本清洗与标准化说明

## 用途

本步骤把数位板原始样本转换为适合用户字体分析和后续写字机轨迹生成的处理数据。处理过程不覆盖、不移动原始样本，所有结果写入 `runtime_data/handwriting/processed/`。

## 默认处理流程

1. 校验坐标、时间、压力和倾角等字段为有限数值。
2. 将样本时间轴从第一个真实落笔点重新归零，去除启动界面后的等待时间。
3. 删除完全重复的传感器事件。
4. 为几何轨迹删除距离小于 `0.0005` 的过密点。
5. 对每个笔画执行一次轻度三点平滑，保持首尾端点不变。
6. 按归一化弧长间隔 `0.004` 对每个笔画重新采样。
7. 生成保留用户原始字面大小和位置的 `canvas` 表示。
8. 生成等比例缩放、居中并保留 10% 边距的 `glyph` 表示。

原始笔画顺序和笔画数量不会改变，也不会合并或拆分笔画。

## 三套轨迹表示

### `dynamics`

只删除完全相同的重复事件，保留时间、压力和倾角变化。用于分析：

- 书写速度；
- 落笔和收笔压力；
- 压力变化曲线；
- 倾角和握笔姿态；
- 笔画时长和停顿。

即使笔尖坐标暂时不动，只要压力或时间发生变化，该点仍会保留。

### `canvas`

经过几何去重、平滑和等弧长重采样，但仍使用原始 `[0, 1]` 画布坐标。用于分析：

- 用户实际字面大小；
- 字符重心和位置偏移；
- 原始宽高比；
- 不同字符的自然尺寸变化。

### `glyph`

把 `canvas` 轨迹等比例缩放并居中到统一方框。用于分析：

- 笔画形态；
- 横竖撇捺的角度与曲率；
- 部件比例；
- 不同字符间可比较的结构特征；
- 后续标准骨架的个人风格变换。

不能只保存 `glyph`，否则会丢失用户原始大小和重心风格。

## 运行方法

在仓库根目录运行：

```powershell
src\handwriting_robot_layer2_wl_handwriting\tools\run_preprocessing.cmd user_01
```

默认输出目录：

```text
runtime_data/handwriting/processed/user_01/style_probe_v1/
```

需要调整参数时：

```powershell
src\handwriting_robot_layer2_wl_handwriting\tools\run_preprocessing.cmd `
  user_01 `
  --duplicate-distance 0.0005 `
  --resample-spacing 0.004 `
  --smoothing-passes 1 `
  --glyph-margin 0.1
```

不建议在没有对比结果的情况下增加平滑次数。过度平滑会改变转折、顿笔和个人笔画特征。

## 输出结构

每个字符生成：

```text
processed/user_01/style_probe_v1/U+4E00/v01.json
```

每个文件包含：

- 原始样本规范化 JSON 内容的 SHA-256，用于追踪处理结果对应的输入；
- 处理参数和点数变化；
- 被去除的等待时间、重复事件和过密几何点数量；
- 原始与清洗后的轨迹长度；
- `dynamics`、`canvas`、`glyph` 三套表示；
- `canvas` 包围盒和 `glyph` 标准化变换参数。

批处理报告位于：

```text
processed/user_01/style_probe_v1/preprocessing_report.json
```

## 非破坏性与隐私

- 不修改 `writers/<writer_id>/samples/` 下的原始样本。
- 重复运行会根据当前原始样本和参数重新生成处理结果。
- 原始样本、处理结果和报告均位于被 Git 忽略的 `runtime_data/`。
- 不得把真实笔迹或处理后的个人特征提交到公共仓库。

## 完成条件

- 100 个样本全部成功处理；
- 没有缺失或失败样本；
- 原始笔画数与处理后笔画数一致；
- 所有处理点为有限值且时间单调不下降；
- `glyph` 坐标位于 `[0, 1]` 并保持原宽高比；
- 抽查简单字和复杂字，平滑后没有明显削平转折或改变字形。

完成后可进入用户字体风格特征提取。
