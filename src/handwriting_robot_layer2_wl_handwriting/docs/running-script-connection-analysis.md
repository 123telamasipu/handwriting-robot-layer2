# 行书连接候选分析说明

## 用途

该工具分析用户真实数位板样本中相邻落笔段的空间和时间关系，为后续研究“哪些楷书笔画可能在该用户的行书中连写”提供只读候选数据。

工具不会连接任何笔画，不生成新的落笔路径，也不输出写字机控制命令。候选结果必须经过字符结构对齐和人工检查后，才能进入后续生成研究。

## 分析内容

对每个相邻实际落笔段边界提取：

- 前一段终点与后一段起点的归一化距离；
- 两段之间的离笔停顿时间；
- 前一段收笔方向到候选连接方向的转角；
- 候选连接方向到后一段起笔方向的转角；
- 用户实际落笔段数与规范楷书笔画数的关系。

默认候选阈值为：端点距离不超过 `0.12`、离笔停顿不超过 `120 ms`、收笔转角不超过 `60°`、起笔转角不超过 `90°`。四项必须同时满足。阈值只是第一版保守筛选条件，不代表书法规则。

## 运行方法

在仓库根目录运行：

```powershell
src\handwriting_robot_layer2_wl_handwriting\tools\run_connection_analysis.cmd `
  runtime_data\handwriting\processed\user_01\style_probe_v1 `
  runtime_data\handwriting\reports\user01_connection_candidates_v1.json
```

报告继续保存在被 Git 忽略的 `runtime_data/`，因为其中包含个人书写行为和真实轨迹端点。

当前 `user_01` 的 100 字报告包含 601 个相邻落笔段边界，默认阈值筛出 51 个候选，候选率约为 `8.5%`。其中只有 5 个达到高置信筛选条件，分别位于“室、差、家、安、江”。它们只是下一步人工语义对齐的优先检查项，不是可执行的连笔规则。

## 输出

- `script_behavior`：实际落笔段少于或多于规范楷书笔画数的字符清单；
- `boundary_statistics`：全部相邻落笔段边界的距离、停顿和方向统计；
- `candidates`：按候选分数排列、等待人工复核的边界；
- `samples`：每个字符的完整边界分析；
- `safety`：明确标记当前为只读分析，不产生落笔连接。

## 重要限制

- 当前分析面对的是“用户实际落笔段 1 → 实际落笔段 2”，尚未完成与规范楷书语义笔画的对应。
- 端点接近且停顿较短不等于书法上允许连写。
- 真实行书可能在一个连续落笔段内部合并多笔；仅分析落笔段边界无法恢复内部楷书笔画分界。
- 不允许把报告中的两个端点直接用直线相连并发送给写字机。

下一步应先对高频偏旁和简单字符建立“用户实际落笔段 ↔ 规范笔画”的人工小规模对齐，再学习连接路径的曲率和速度。
