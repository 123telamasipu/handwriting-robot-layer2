# 成员2功能集成与其他成员配合清单

## 成员2当前交付

成员2模块已经形成以下主链路：

```text
数位板样本
  → 非破坏性预处理
  → 用户整体风格分析
  → 行书落笔段自动对齐
  → 个人字体档案
  → 多种子批量质量评估
  → 逐字增强/回退部署策略
  → StrokeDocument
  → 写字机联调轨迹交付包
```

成员2只负责生成 `StrokeDocument`，不负责正式页面排版、设备通信或写字机控制。

公开代码、命令行入口、测试及格式说明可以提交 Git。真实笔迹、个人字体档案、评估报告、部署策略、Hanzi Writer 数据和联调轨迹均位于被忽略的 `runtime_data/`，不得提交或直接分享到公开仓库。

## 成员1：系统集成与页面任务

### 需要接收

- 成员2生成的 `StrokeDocument` v0.1；
- 联调包中的 `handoff_manifest.json` 和 `stroke_documents/*.json`；
- `generation.characters` 中的字符、笔画序号和实际生成策略；
- `motion_hints`，但它只能作为用户书写节奏参考。

### 需要完成

1. 将成员2的模块局部轨迹放入最终页面布局，不直接修改成员2的私有个人字体产物。
2. 按统一接口补充页面任务所需的 `origin_mm`、`scale`、`rotation_deg` 和 `region`。
3. 完成换行、页边距、区域裁切、图文混排及多模块笔画排序。
4. 检查页面变换后所有坐标仍处于最终书写区域和 A4 页面范围。
5. 将排版后的有序轨迹交给设备模块；不能把成员2的 `motion_hints` 直接当作设备命令。
6. 冻结或升级统一接口时，同步通知成员2更新示例和测试。

### 需要回传给成员2

- 最终确认的 `StrokeDocument`/页面任务字段版本；
- 页面坐标原点和方向约定；
- 页面排版允许的最小/最大字符尺寸；
- 集成层对最大轨迹点数、文件大小和字符数的限制；
- 联调失败时的具体输入文件、错误字段和复现步骤。

## 成员5：设备、标定与真实落笔测试

### 需要接收

- 已通过软件预检但明确标记 `machine_ready=false` 的联调包；
- 五类场景：字符级增强、高/中置信评估回退、低置信回退、未采字符和混合文本；
- 每份轨迹的点数、笔画数、包围盒、最大相邻点距和建议时长统计。

### 需要完成

1. 建立页面毫米坐标到机器坐标的标定映射并记录标定误差。
2. 检查机械行程、限位、夹纸机构、笔架高度和抬笔安全距离。
3. 确定不同笔和纸张下可用的速度、加速度、转角减速和抬笔参数。
4. 先进行不落笔空跑和急停验证，再从低速、单字符、废纸开始落笔。
5. 分别测试增强、两类回退和未采字符，记录可读性、尺寸误差、漂移、振动和漏笔。
6. 设备测试结果应写入设备模块或 `docs/test-records/`，不要把真实用户笔迹原始文件复制到公开目录。

### 需要回传给成员2

- 设备能够稳定处理的最大相邻点距和建议重采样间距；
- 最小有效线段长度、最大单任务点数和最大单页笔画数；
- 经过验证的字符尺寸范围和页面安全边距；
- 速度、加速度或转角处出现抖动的具体轨迹文件及笔画序号；
- 真实落笔照片或误差数据的脱敏评估结论；
- 是否需要成员2调整轨迹采样密度、字符尺寸或曲线强度。

成员5负责决定设备参数是否安全。成员2不会生成串口命令、G-code、固件指令或直接启动设备。

## 成员4：界面调用

### 需要完成

1. 界面只向集成层提交 `user_id`、文字、随机种子和生成选项，不直接读取或修改私有样本文件。
2. 在界面中区分“个人字体档案可用”“部署策略已验证”和“设备可执行”三个状态，不能把软件生成成功显示为机器已安全就绪。
3. 展示明确错误：缺少骨架、部署哈希失效、页面越界或个人档案不存在。
4. 暂停、恢复和开始书写仍由集成/设备层管理，不能在界面中复制成员2或成员5逻辑。

### 需要回传给成员2

- 界面需要稳定支持的请求字段；
- 用户可调整参数的范围和默认值；
- 失败提示所需的错误码或结构化信息需求。

## 成员3：图形模块接口一致性

成员3不依赖个人字体档案，但双方需要继续使用相同的 `StrokeDocument` v0.1 坐标方向、单位、笔画顺序和 `pen_down` 语义。若图形模块或集成层提出接口变更，应由成员1统一评审，避免文字和图形各维护一套轨迹格式。

## 推荐联调顺序

1. 成员1读取 `handoff_manifest.json`，完成页面任务包装和区域变换。
2. 成员1、成员5只用 `mixed_smoke` 做不落笔空跑。
3. 成员5确认限位、急停和标定后，依次低速测试单场景轨迹。
4. 成员5回传机械限制和误差，成员2只调整轨迹生成或软件预检参数。
5. 成员1组织完整的文字、图形、界面和设备集成测试并记录版本。

## 当前阻塞项

- 正式页面任务中的 `origin_mm`、`scale`、`rotation_deg`、`region` 尚需成员1实现和确认；
- 设备坐标标定、速度/加速度、笔压、空跑和急停审核尚需成员5完成；
- 在上述两项完成前，联调包不能标记为机器可执行；
- 字母、数字和符号仍需后续接入许可清晰的单线骨架数据源。
